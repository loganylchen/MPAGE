# AGENTS.md - Guidelines for AI Agents

> This file provides guidelines for AI coding agents working on the MPAGE R package.

## Project Overview

**MPAGE** is an R package for Module-based Pair-wise Analysis of Gene Expression. It provides workflows for analyzing RNA modification modules using PPI networks and gene expression data.

## Build/Lint/Test Commands

### Testing
```bash
# Run all tests
devtools::test()

# Run a single test file
devtools::test_file("tests/testthat/test-rna_mod_proteins.R")

# Run tests matching a pattern
devtools::test(filter = "rna_mod_proteins")
```

### Build & Check
```bash
# Build the package
devtools::build()

# Run R CMD check
devtools::check()

# Quick check (no vignettes)
devtools::check(args = "--no-vignettes")
```

### Documentation
```bash
# Generate roxygen2 documentation
devtools::document()

# Build vignettes
devtools::build_vignettes()
```

### Installation
```bash
# Install from source locally
devtools::install()

# Install from GitHub
devtools::install_github("loganylchen/MPAGE")
```

## Code Style Guidelines

### Function Naming
- Use `snake_case` for all function names (e.g., `get_rna_mod_proteins`, `identify_modules_iterative`)
- Exported functions should be verbs describing actions
- Internal helper functions should start with `.` (e.g., `.get_built_in_rna_mod_proteins`)

### Variable Naming
- Use `snake_case` for variables (e.g., `expression_data`, `sample_condition`)
- Use descriptive names; avoid single-letter variables except in iterators
- Boolean variables should start with `is_`, `has_`, or `use_` (e.g., `use_built_in`)

### Indentation & Formatting
- Use **2 spaces** for indentation (not tabs)
- Maximum line length: 80-100 characters
- Place opening braces on same line: `function() {`
- Always use explicit returns at end of functions

### Imports & Dependencies
- Use `importFrom` in NAMESPACE (generated by roxygen2 `@importFrom` tags)
- Prefer importing specific functions over entire packages
- Use `::` for function calls when not imported (e.g., `igraph::V()`)
- Core packages: igraph, STRINGdb, clusterProfiler, dplyr, ggplot2, etc.

### Documentation (roxygen2)
Every exported function must have:
- `#' Title` - Brief description
- `#' Description` - Detailed explanation
- `@param` - Document each parameter with type and description
- `@return` - Describe return value structure
- `@export` - For exported functions
- `@examples` - Include runnable examples in `\dontrun{}` if needed
- `@noRd` - For internal functions

Example:
```r
#' Get RNA Modification Proteins
#'
#' Retrieve curated list of RNA modification-related proteins
#'
#' @param use_built_in Logical whether to use built-in data frame
#' @return A data frame containing RNA modification proteins
#' @export
get_rna_mod_proteins <- function(use_built_in = TRUE) {
  # implementation
}
```

### Error Handling
- Use `stop()` for errors with informative messages
- Use `warning()` for recoverable issues
- Use `message()` for informational output (sparingly)
- Wrap external calls in `tryCatch()` when appropriate

Example:
```r
if (!is.matrix(expression_data)) {
  stop("expression_data must be a matrix or data frame")
}

tryCatch({
  result <- some_operation()
}, error = function(e) {
  warning("Operation failed: ", e$message)
  return(NULL)
})
```

### Testing (testthat)
- Test files in `tests/testthat/` matching pattern `test-*.R`
- Use `test_that("descriptive test name", { ... })`
- Common expectations: `expect_true()`, `expect_equal()`, `expect_silent()`
- Use temporary files with `tempfile()` and clean up with `unlink()`

Example:
```r
test_that("function returns correct structure", {
  result <- my_function()
  expect_true(is.data.frame(result))
  expect_equal(ncol(result), 4)
})
```

### Data Handling
- Store package data in `data/` directory
- Store raw data processing scripts in `data-raw/`
- Use `.rds` format for internal data files in `inst/extdata/`
- Set seed for reproducibility: `set.seed(422)`

### Package Structure
```
MPAGE/
├── R/              # R source code
├── tests/          # Test files (testthat)
├── data/           # Package datasets
├── data-raw/       # Data preparation scripts
├── man/            # Rd documentation (auto-generated)
├── vignettes/      # Vignettes
├── inst/extdata/   # External data files
├── DESCRIPTION     # Package metadata
└── NAMESPACE       # Export list (auto-generated)
```

### Git Workflow
- Never commit `.Rproj.user/`, `temp/`, or `main_temp/` directories
- Never commit large data files (use .gitignore)
- Keep commits focused and atomic

## Key Conventions

1. **Purity**: Functions should be pure when possible; avoid global state
2. **Validation**: Validate inputs early with clear error messages
3. **Progress**: Use `message()` to report progress in long-running operations
4. **Reproducibility**: Always set random seeds in functions using randomization
5. **Bioconductor**: Follow Bioconductor conventions for genomics data structures
